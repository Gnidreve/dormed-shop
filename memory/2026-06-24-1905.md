**Dein Ansatz (`/test/index`):** Eigene Routen, gleiche Views, andere Credentials — korrekte Intuition, aber tatsächlich viel Duplicate Code bei Routes + Controllers.

**Sauberer Gegenvorschlag: `SandboxMiddleware` + URL-Prefix**

```
/          → Live-Modus (Stripe/PayPal-Credentials aus DB, Live)
/test/*    → Sandbox-Modus (gleicher Code, andere Config-Schicht)
```

**Wie es funktioniert — ohne Duplicate Code:**

Eine Middleware `ForceTestMode` hängt sich vor alle `/test/*`-Routen:
- Setzt `payment.provider.mode = sandbox` für den aktuellen Request (ohne DB zu ändern)
- Setzt `app.test_checkout = true` im Request-Kontext
- Alle Controller, Views, Services bleiben 1:1 gleich

Orders in Testmodus bekommen `is_test = true` in der DB — im Admin separat filterbar.

**PayPal macht es genauso:** `api-m.sandbox.paypal.com` vs `api-m.paypal.com` — gleiche API-Struktur, komplett andere Endpoint-URL. Unsere `PayPalService::buildConfig()` baut schon den richtigen URL je nach Mode. Middleware würde einfach `mode = sandbox` erzwingen.

**Vorteile:**
- Kein doppelter Code
- Sandbox-Bestellungen sind klar markiert in der DB
- `/test` kann jederzeit offen bleiben ohne Einfluss auf Live
- Einfach zu testen: `/test/checkout/confirm` — identischer Flow, Sandbox-API

In `HandleInertiaRequests.php` gibt es eine `share()`-Methode. Alles was du dort zurückgibst, steht **auf jeder Seite automatisch** zur Verfügung:

```php
// app/Http/Middleware/HandleInertiaRequests.php
public function share(Request $request): array
{
    return [
        ...parent::share($request),
        'auth'     => [...],
        'cart'     => $cart,
        'testMode' => config('app.test_mode', false),  // ← hier
    ];
}
```

Inertia serialisiert das als JSON in den initialen HTML-Response (`<div id="app" data-page="{...}">`). Das ist die "Injection".

**Schritt 2 — Svelte liest es aus (JS)**

```svelte
<!-- TestModeBanner.svelte -->
<script lang="ts">
    import { page } from '@inertiajs/svelte';
</script>

{#if page.props.testMode}
    <div class="bg-amber-400 ...">Testmodus aktiv</div>
{/if}
```

`page` ist ein Svelte-Store von Inertia — reaktiv, kein manuelles Fetching nötig. `page.props` ist das direkte Abbild von `share()`.
